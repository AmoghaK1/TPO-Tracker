# Contributing to TPO-Tracker

Thank you for your interest in contributing to TPO-Tracker! We're excited to have you join our community and help make this project better.

## Getting Started

Before you start contributing, please **read the [README.md](README.md)** carefully. It contains important information about:
- Project setup and installation
- Prerequisites and dependencies
- Configuration requirements
- How the bot works

A proper local setup is essential before contributing any code.

## Project Goals

TPO-Tracker is an open-source initiative with two primary objectives:

1. **Streamline the Placement Portal Experience for VIT Students**
   - Automate the tedious manual checking of the TPO portal
   - Provide instant notifications when new companies are added
   - Reduce time spent monitoring placement opportunities

2. **Expand Accessibility to Multiple Educational Institutions**
   - Create a scalable solution that can be adapted for other colleges
   - Build a foundation that supports different portal structures
   - Enable students across institutions to benefit from automated placement tracking

## How to Contribute

There are many ways you can contribute to this project:

### 1. Report Issues
If you encounter a bug or have a feature request:
- Check if the issue already exists in the [Issues](../../issues) section
- Provide a clear, descriptive title
- Include steps to reproduce (for bugs)
- Share relevant error messages or logs
- Mention your environment (Windows version, Python version, etc.)

### 2. Fix Issues
Help us resolve existing issues:
- Look for issues tagged with `good first issue` or `help wanted`
- Comment on the issue to let us know you're working on it
- Follow the submission guidelines below

### 3. Implement Features
Have an idea to improve TPO-Tracker?
- Open an issue first to discuss your proposal
- Get feedback from the community
- Wait for approval before starting development
- This prevents duplicate work and ensures alignment with project goals

### 4. Improve Documentation
Documentation is crucial:
- Fix typos or unclear explanations in README.md
- Improve code comments
- Add troubleshooting tips
- Document new features

## Submission Guidelines

### Before You Start
1. Fork the repository
2. Create a new branch with a descriptive name: `feature/your-feature` or `fix/issue-description`
3. Set up your development environment following the README

### Code Requirements
- Follow PEP 8 Python style guidelines
- Write clear, self-documenting code with meaningful variable names
- Add comments for complex logic
- Test your changes locally before submitting

### Commit Messages
Write clear, concise commit messages:
- ❌ `fix bug`, `update`, `changes`
- ✅ `Fix chromedriver timeout issue on slow networks`, `Add retry logic for Telegram notifications`

### Pull Request Process
1. Push your changes to your forked repository
2. Open a pull request against the `main` branch
3. Provide a clear description of:
   - What changes you made
   - Why you made them
   - How to test the changes
4. Reference any related issues using `#issue-number`
5. Wait for review and address any feedback

### Adapting for Other Colleges
If you're modifying TPO-Tracker for another institution:
- Create a feature branch (e.g., `college/college-name-portal`)
- Document portal-specific changes clearly
- Abstract college-specific logic where possible
- Open a pull request with detailed notes on the portal structure

## Community Guidelines

- Be respectful and constructive in all interactions
- Provide helpful feedback when reviewing others' work
- Ask questions if something is unclear
- Share knowledge and help other contributors
- Acknowledge and credit collaborators

## Questions or Need Help?

- Check the README troubleshooting section
- Review existing issues and discussions
- Feel free to ask in an issue or pull request

## Recognition

All contributors will be acknowledged in the project. We value every contribution, whether it's code, documentation, bug reports, or ideas.
